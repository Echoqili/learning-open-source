## Skills Manager Web 应用全面测试与优化

### 发现的 Bug

1. **后端核心 Bug**：`build_skills_cache()` 读取 `data.get("skills", [])` 错误，实际 `skills-index.json` 结构为 `{"sources": {"learning-open-source": [...], ...}}` 字典格式，导致返回 0 个 skills。

2. **分类字段缺失**：skills-index.json 中的 skill 对象只有 `name`、`path`、`description`、`content` 字段，没有 `category`、`category_emoji`、`category_name`，需要从路径提取。

3. **`/api/category/<cat_key>` 参数错误**：使用了 `by_name` 变量（第3个返回值）当作 by_category，实际应为 `by_category`。

4. **`/api/skill/<name>` 详情返回 `skill.emoji` 未定义**：前端访问 `skill.emoji` 时为 undefined。

5. **`/api/scenario/<key>` 无效**：只返回 `name` 和 `emoji`，没有实际 skills 数据。

6. **前端初始化为空**：首屏无内容，没有加载任何 skills。

### 新增：中英文切换功能

**实现方式**：
- 页面右上角固定语言切换按钮（中/EN）
- `data-i18n` 属性标记静态 HTML 元素，`applyI18n()` 统一更新
- `t(key, ...args)` 函数用于 JS 动态生成的文案
- 翻译字典含 zh/en 各 150+ 个 key，覆盖所有 UI 文案
- `refreshCurrentView()` 在语言切换时重新渲染当前视图
- `localStorage` 持久化语言偏好
- 原始 skill 数据（名称、描述、文件名）始终保持原文，不被翻译

**修改文件**：`web/templates/index.html`
- CSS: 添加 `.lang-toggle`、`.lang-menu` 样式
- HTML: 语言切换按钮 + `data-i18n`/`data-i18n-placeholder` 属性
- JS: 完整翻译字典、`t()`/`applyI18n()`/`setLang()`/`refreshCurrentView()` 函数
